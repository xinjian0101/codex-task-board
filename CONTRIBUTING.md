# Contributing

Read the README, About, and Roadmap before opening a change.

Use an Issue to describe large changes. Keep each pull request focused and include the problem, intended result, validation steps, and documentation updates.

Use clear commit prefixes such as `feat:`, `fix:`, `docs:`, `test:`, `refactor:`, and `chore:`.

Contributors are responsible for correctness, licensing, and review of submitted material. Be respectful and do not publish private information.
