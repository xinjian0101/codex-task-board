# Security Policy

This project is in the foundation phase and has not published a production-ready release.

Do not post private project files or sensitive technical details in public Issues. Use GitHub private reporting when available. Otherwise open a minimal Issue requesting a private communication channel.

A report should identify the affected commit, environment, reproduction steps, observed impact, and a sanitized example.

Validated reports will be reviewed privately where practical. Public notes should explain affected versions and corrections without exposing user information.
